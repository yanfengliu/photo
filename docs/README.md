# Docs Index

This directory contains the canonical project documentation.

## Sections

- `architecture/` - system overview, architectural decisions, and drift history.
- `devlog/` - compact summary plus dated detailed task logs.
- `learning/` - short, durable lessons worth keeping.
- `debugging/` - templates and notes for debugging sessions.
- `reviews/` - review artifacts and short reviewer notes when they are useful to keep.

## Legacy paths

The flat files at the root of `docs/` are temporary compatibility shims for older references.
Do not add new content there.

## Canonical Edit Targets

Edit only these locations for durable documentation updates:

- `docs/architecture/`
- `docs/devlog/`
- `docs/learning/`
- `docs/debugging/`
- `docs/reviews/`
